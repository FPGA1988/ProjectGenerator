# 1 启动文件
启动文件名为syn_digital，本文件需要在shell里面进行调用。它的基本功能如下 ：
- 更改运行文件夹  
- 删除旧的临时文件    
- 将主程序脚本传入DC  
- 记录DC运行日志  

## 1.1 运行路径
在DC运行过程中，会产生包括*.mr,*.pvl以及*.syn的临时文件，这些临时文件可能会影响运行结果，因此在调用DC之前，先运行以下脚本
将这些文件给删除 : 
```Shell
cd ../run_dir #跳转路径到临时运行文件夹
rm *.pvl
rm *.syn
rm *.mr  
```
除了以上文件之外，它还有可能会存在文件夹`alib-52`,alib应该是一类库，这个等待后续分析.

## 1.2 启动DC
在删完临时文件之后，调用以下命令启动dc_shell : 
`dc_shell -f ../scripts/run_digital.tcl | tee ../log/digital.log`  
`-f`参数用于指定可执行命令文件，这一选项是可选的，run_digital.tcl就是要运行的主程序/脚本.可以运行`dc_shell -help`调用帮助
文，添加`-gui`参数可以运行gui.

### 1.2.1 Shell管道'|'  
在dc后面，使用了一个管道`|`，所谓管道channel，则是将**上一个命令的标准输出送到下一个命令的标准输入**，而`tee`则是将标准
输入读到的数据输出到标准输出并且保存到文件中。如果这里使用重定向`>`或者`>>`，则不会在标准输出上显示。这样，使用`tee`则可以
将dc的log保存到log文件，方便后续查看。

# 2 主程序/脚本  
主程序run_digital.tcl是综合的核心脚本，它会建立起dc的运行环境，并对整个设计进行综合、时序分析、网表输出等。它主要包括三个
部分 : 配置、约束以及综合，下面将分别对这三个部分进行详细的说明，在这之前，首先进行一些基本的环境配置。

## 2.1 运行环境配置
运行环境主要先创建一些需要的文件夹等，运行的脚本有 :  
```
sh date
sh hostname
set SCRIPTS_DIR	"scripts"
set REPORTS_DIR	"reports"
set RESULTS_DIR	"results"
```
### 2.1 sh命令  
首先要认识到，主脚本已经是运行于dc_shell中，里面所有的命令都是支持于dc，而和外面的环境已经没有关系了，`sh`就是一个dc的内部
命令。
> 名称 : sh	在一个子进程中执行一条命令
> 语法 : sh [args]
> 说明 : 这个和exec命令非常相似，但是它的参数支持文件名扩展.包含空格的参数需要使用双引号""或者大括号{}包含起来.

> 示例 : sh rm aaa* [*会进行匹配]
